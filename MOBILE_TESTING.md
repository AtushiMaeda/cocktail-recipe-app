# スマホシェイク機能の実機確認手順

このドキュメントでは、開発環境でスマホの実機を使ってシェイク機能をテストする方法を説明します。

## 前提条件

- Device Motion APIはHTTPS環境でのみ動作します
- スマホとPCが同じWiFiネットワークに接続されている必要があります

## 推奨方法: Vite自己署名証明書を使用したHTTPSテスト

### 1. HTTPS開発サーバーの起動
```bash
npm run dev:https
```

### 2. ネットワークアドレスの確認
開発サーバー起動時に表示されるHTTPS アドレスを確認：
```
Local:   https://localhost:5173/
Network: https://192.168.1.100:5173/  ← このアドレスを使用
```

### 3. スマホからアクセス
1. スマホのブラウザで `https://[表示されたIPアドレス]:5173/` にアクセス
2. **重要**: 自己署名証明書の警告が表示されるので「詳細設定」→「安全でないサイトに進む」を選択
3. SSL証明書の警告を受け入れる

## 代替方法1: HTTPテスト（機能制限あり）

Device Motion APIは制限されますが、基本的な動作確認は可能です。

### 1. HTTP開発サーバーの起動
```bash
npm run dev:mobile
```

### 2. スマホからアクセス
スマホのブラウザで `http://[表示されたIPアドレス]:5173/` にアクセス

**注意**: HTTPではDevice Motion APIが動作しないため、シェイク機能は使用できません。

## 代替方法2: ngrokを使用したHTTPSテスト

ngrokアカウントを持っている場合の手順です。

### 1. ngrokの認証設定
```bash
ngrok config add-authtoken YOUR_AUTHTOKEN
```

### 2. 開発サーバーの起動
```bash
npm run dev
```

### 3. ngrokでトンネルを作成
別のターミナルウィンドウで：
```bash
ngrok http 5173
```

### 4. HTTPS URLでアクセス
ngrokが表示するHTTPS URL（例：`https://abc123.ngrok.io`）をスマホのブラウザでアクセス

## テスト手順

### 1. アプリへのアクセス
スマホのブラウザでアプリにアクセスします。

### 2. シェイク機能の有効化
1. 「シェイクOFF」ボタンをタップ
2. iOS Safariの場合、権限許可ダイアログが表示されるので「許可」を選択
3. ボタンが「シェイクON」に変わることを確認

### 3. シェイクテスト
1. スマホを上下・左右・前後に振る
2. シェイクアニメーション（🍹マークの回転）が表示される
3. ランダムなカクテルが表示される
4. 対応端末ではバイブレーションが発生する

### 4. 機能確認項目
- [ ] シェイクON/OFF切り替えが動作する
- [ ] iOS Safariで権限リクエストが表示される
- [ ] スマホを振るとアニメーションが表示される
- [ ] ランダムなカクテルが選択される
- [ ] 連続シェイクが防止される（1秒間のタイムアウト）
- [ ] バイブレーション機能が動作する（対応端末のみ）

## トラブルシューティング

### シェイクが反応しない場合
- **HTTPS環境か確認**: DevToolsのコンソールでエラーを確認
- **自己署名証明書を受け入れ**: スマホブラウザで警告を受け入れる
- **権限が許可されているか確認**: iOSの場合は特に注意
- **振り方を変える**: 上下・左右・前後など異なる方向で試す
- **閾値の調整**: 必要に応じて `SHAKE_THRESHOLD` を調整

### アクセスできない場合
- **ファイアウォール設定**: PCのファイアウォールを確認
- **ネットワーク設定**: 同じWiFiネットワークに接続されているか確認
- **ポート設定**: 5173番ポートが開いているか確認
- **証明書エラー**: 自己署名証明書の警告を受け入れる

### 自己署名証明書の警告対処法

#### iOS Safari
1. 「このサイトは安全ではありません」の警告で「詳細」をタップ
2. 「この Web サイトにアクセス」をタップ

#### Android Chrome
1. 「接続がプライベートではありません」の警告で「詳細設定」をタップ
2. 「[IPアドレス]にアクセスする（安全ではありません）」をタップ

### iOS Safariでの注意点
- iOS 13+では`DeviceMotionEvent.requestPermission()`が必要
- HTTPでは権限が取得できない（HTTPSが必須）
- プライベートブラウズモードでは制限がある場合がある
- 自己署名証明書の警告を必ず受け入れる

### Android Chromeでの注意点
- 権限リクエストは不要（自動で有効）
- HTTPSでない場合、機能が無効化される場合がある
- 自己署名証明書の警告を受け入れる必要がある

### ngrokエラーの対処法
ngrokで認証エラーが発生した場合：
1. [ngrok公式サイト](https://dashboard.ngrok.com/signup)でアカウント作成
2. authtokenの設定：`ngrok config add-authtoken YOUR_TOKEN`
3. 代替として**Vite自己署名証明書**の使用を推奨

## 設定詳細

### Vite設定 (vite.config.ts)
```typescript
export default defineConfig({
  server: {
    host: true, // 外部からのアクセスを許可
    https: false, // 必要に応じてHTTPS有効化
    port: 5173,
  },
})
```

### シェイク検出設定
- **閾値**: 25 (調整可能)
- **タイムアウト**: 1000ms
- **検出方式**: 加速度センサーの差分計算

## 本番環境での注意点

本番環境では必ずHTTPS環境でデプロイしてください。HTTP環境ではDevice Motion APIが動作しません。
